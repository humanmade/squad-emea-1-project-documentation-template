# Integrations

**Summary.** External services the project depends on. One child page per integration (analytics, CRM, search index, payment, marketing automation, etc.).

## Per-integration template

Each integration page covers:

- Purpose and which features rely on it.
- Auth model — where credentials live, who has access.
- Data flow — what we send and receive, frequency.
- Vendor contact and support channel.
- Runbook — key rotation, what to do if it goes down.
